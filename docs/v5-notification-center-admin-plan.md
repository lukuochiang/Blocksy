# Blocksy V5 通知中心后台实施计划

更新时间：2026-04-09

## 1. 目标范围

本阶段只做通知中心后台高价值能力：

1. 公告列表（分页/筛选）
2. 公告撤回
3. 公告二次下发
4. 指标趋势图（7天/30天）

## 2. 后端改造计划（blocksy-server）

### 2.1 API（新增/补强）
- `GET /api/admin/notifications/announcements`
  - 参数：`keyword`、`status`、`page`、`pageSize`
  - 返回：分页列表（公告标题、内容摘要、状态、创建时间、发送量）
- `POST /api/admin/notifications/{id}/revoke`
  - 说明：公告状态改为撤回，前台不再展示
- `POST /api/admin/notifications/{id}/redispatch`
  - 说明：对已发布公告执行再次下发，写入通知记录
- `GET /api/admin/notifications/trend`
  - 参数：`days=7|30`
  - 返回：按日聚合的发送量/已读量/已读率

### 2.2 数据层建议
- 复用现有通知表为主，必要时补充公告主表（若当前未区分系统公告源）。
- 趋势统计优先走 SQL 聚合，避免先引入时序组件。

## 3. 管理后台改造计划（blocksy-admin）

### 3.1 页面
- 新增通知中心管理页：
  - 列表区：筛选 + 分页
  - 操作区：撤回、二次下发
  - 统计区：7/30天趋势图卡片

### 3.2 交互
- 撤回：二次确认弹窗 + 操作结果提示。
- 二次下发：可选“立即下发”，返回新增下发数量。
- 趋势：默认 7 天，可切换 30 天。

## 4. 验收标准

1. 后台可分页查询公告列表。
2. 撤回后公告不再作为有效公告对外展示。
3. 二次下发后通知总量有明确增长。
4. 7/30 天趋势图与后端统计数据一致。

## 5. 实施顺序

1. 后端接口与聚合统计先完成并通过 Swagger 验证。
2. 后台页面接接口，先保障功能正确，再优化视觉细节。
3. 补一条通知中心 smoke 脚本，加入 CI 第二条 workflow 或并入现有 workflow 矩阵。

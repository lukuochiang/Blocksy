# Blocksy V6 下一批业务扩展计划（草案）

更新时间：2026-04-10

## 0. 目标

在已完成 V5 Batch 5~9 的基础上，进入下一批“可运营增强”，优先做对 MVP 落地价值最高的闭环。

## 1. 本轮先做（P0）

### 1.1 分类信息扩展闭环

- 前台：
  - 详情页增强（联系信息、状态时间线、相关推荐占位）
  - 我的发布支持按状态筛选与快速上下架
- 后台：
  - 审核状态流转批量化（通过/驳回/下架/恢复）
  - 审核备注模板联动（统一治理动作文案）
- 数据：
  - 分类维度统计（按社区/分类/状态）

### 1.2 活动治理强化

- 前台：
  - 下架活动详情降级提示统一
  - 报名入口不可用态统一（下架/已满/已结束）
- 后台：
  - 批量处理失败项重试 + 导出
  - 处理日志按动作/操作人/时间筛选

### 1.3 通知中心稳定化

- 统一触发口径：
  - 审核结果通知
  - 活动报名通知
  - 系统公告通知
- 指标：
  - 7/30 天触达、已读、未读趋势

## 2. 工程保障（P0）

- CI 增补：
  - 分类信息扩展 smoke
  - 活动治理 smoke
- 回归脚本：
  - 把批量处理 + 导出链路并入统一回归
- 文档：
  - 更新模块状态与接口清单

## 3. 建议排期

- Day 1-2：分类信息后端批量审核 + 统计接口
- Day 3-4：admin 列表批量操作 + 统计卡片联调
- Day 5：app 详情与我的发布状态操作
- Day 6：活动治理重试与导出增强
- Day 7：通知链路复核 + smoke/文档收口

## 4. 验收标准

- 分类信息：前后台从发布到审核到下架恢复全链路可联调
- 活动治理：批量失败可重试，导出结果可复盘
- 通知：三类通知触发稳定，统计接口与页面一致
- 工程：CI smoke 全绿

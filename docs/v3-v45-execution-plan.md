# Blocksy V3 ~ V4.5 开发执行计划（已开工）

更新时间：2026-04-09

## 0. 开发原则

1. 每个版本先打通后端接口 + 前台最小可联调页面 + 后台管理入口。
2. 每个版本都要有回归脚本或可复用联调脚本。
3. 不引入微服务、MQ、ES，保持模块化单体。

## 1. V3（活动闭环增强）

目标：活动发布、列表、详情、报名、后台治理闭环稳定。

### 已开工
- 活动批量处理与日志导出接口已可用。
- 备注模板已接入活动治理使用链路。

### 下一批开发项
1. [已完成] 前台活动详情降级态统一（下架/删除活动说明、报名按钮禁用态）
2. [已完成] 活动我的报名页增强（状态标签、时间过滤）
3. [已完成] 后台活动治理动作补充理由模板选择联动

## 2. V3.5（运营与治理深化）

目标：治理动作标准化、审计可追踪、效率工具化。

### 下一批开发项
1. [已完成] 治理动作模板化（按模块+动作维度）
2. [已完成] 操作日志筛选维度补齐（操作者、时间、动作、对象）
3. [已完成] 批量治理失败项重试能力（活动/举报）

## 3. V4（通知中心）

目标：评论通知、活动通知、系统通知可见可读可统计。

### 下一批开发项
1. [已完成] 通知分类分页（评论/点赞/活动/系统）
2. [已完成] 已读未读状态批量切换能力
3. [已完成] 后台系统公告发布->用户通知下发闭环

## 4. V4.5（体验与数据化）

目标：通知触达与治理效率数据化，支持运营复盘。

### 下一批开发项
1. [已完成] 通知触达统计（发送量、已读率）
2. [已完成] 治理面板关键指标（举报处理时效、重复违规用户）
3. [已完成] 回归脚本整合（`scripts/regression_v35_v45_flow.sh`）

## 5. 交付节奏建议（两周）

1. 第 1 周：完成 V3 收口 + V3.5 核心治理工具
2. 第 2 周：完成 V4 通知中心 + V4.5 指标与回归整合

## 6. 验收门槛

1. 接口有 Swagger 文档且能在本地直接联调。
2. 前台关键页面有空状态/加载状态/错误状态。
3. 后台管理页面支持筛选、批量操作、日志追踪。
4. 每一版至少 1 条自动化回归脚本可执行。
